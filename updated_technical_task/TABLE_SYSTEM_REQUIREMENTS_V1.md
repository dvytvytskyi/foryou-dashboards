# ТЗ ДЛЯ ВНЕШНЕЙ СИСТЕМЫ ПЛАНОВ (Блок: ОБЗОР)
**Проект:** Foryou Dashboards (v2.0)  
**Источник:** Google Таблицы (или база данных)

---

## 1. СТРУКТУРА ЛИСТА «ПЛАНЫ БРОКЕРОВ»

Для расчета рейтинга брокеров (% выполнения плана) нам необходима ежемесячная сводка целей.

| Колонка (Поле) | Тип данных | Пример данных | Обязательность |
| :--- | :--- | :--- | :--- |
| **Месяц** | Дата (ММ.ГГГГ) | 03.2026 | 🔴 Критично |
| **Брокер (ID / Имя)** | Текст | Динара Исаева | 🔴 Должно совпадать с CRM |
| **Цель: Количество сделок** | Число | 5 | 🔴 Критично |
| **Цель: Сумма продаж (AED)** | Число | 15,000,000 | 🟡 Важно |
| **Цель: Доход компании (AED)** | Число | 120,000 | 🟡 Важно |

---

## 2. СТРУКТУРА ЛИСТА «РАСХОДЫ»

Для полноценного расчета **ROMI** и **ROI** (Блок Маркетинг), нам необходимо фиксировать затраты.

| Колонка (Поле) | Тип данных | Пример данных | Источник |
| :--- | :--- | :--- | :--- |
| **Дата** | Дата | 31.03.2026 | 🔴 |
| **Категория расхода** | Список | Офис / Offline Маркетинг | 🔴 |
| **Суб-источник** | Текст | Блогеры / Билборды | 🟢 Только manual расходы |
| **Сумма (AED)** | Число | 5,000 | 🔴 |

---
**Внимание:** Затраты на Facebook Ads, Google Ads и Property Finder в таблицу **НЕ ВНОСЯТСЯ**. Они подтягиваются автоматически через API напрямую в аналитику.

---

## 3. ТРЕБОВАНИЯ К ДАННЫМ
*   **Имена брокеров** должны строго совпадать с их именами в amoCRM (для корректной "склейки" данных). 
*   **Валюта:** Все данные вносятся строго в **AED**. 

---

## 4. СЕГМЕНТАЦИЯ ПЛАНОВ ПО НАПРАВЛЕНИЯМ
Для корректного отображения KPI по департаментам, в таблицу планов необходимо добавить разметку:

| Поле | Опция | Обязательность |
| :--- | :--- | :--- |
| **Департамент** | Off plan / Secondary / Rent | 🔴 Критично |

**Логика:** У каждого брокера может быть индивидуальный план на каждый конкретный департамент в отдельности.

---

## 5. УНИФИКАЦИЯ ИМЕН (MAPPING)
1.  **ФИО Брокера:** Все имена в таблицах должны **символ в символ** совпадать с именами Пользователей в amoCRM. Например: `Григорий Петров` (не сокращать до `Г. Петров`).

---

## 6. ДЕТАЛИЗАЦИЯ ПЛАНОВ ПО БРОКЕРАМ (ПЛАН/ФАКТ)
Для корректного замера % выполнения планов, в таблицу необходимо ежемесячно вносить цели:

| Поле (Лист «Планы») | Тип данных | Роль в системе |
| :--- | :--- | :--- |
| **Цель: Новые Лиды** | Число | Будет сравниваться с фактическими «Received» за месяц. |
| **Цель: Кол-во Сделок**| Число | Будет сравниваться с фактическими «Won» за месяц. |
| **Цель: Выручка (AED)** | Число | Будет сравниваться с фактическим «Net Income» |

> **Формула:** `% Выполнения Лидов = (Фактически Получено) / (Цель по лидам) * 100`.

---
**Примечание:** Для работы формул SLA и First Touch дополнительных данных в таблице не требуется, они тянутся автоматически из CRM.
