# ТЗ: Переход на полную аналитику из amoCRM (v2.0)

Данный документ описывает структуру переноса всех отчетных данных из внешних таблиц напрямую в amoCRM для создания единого источника правдивой аналитики.

### 1. Отдел продаж (New)
*   **Обзор**
    *   К-во сделок: Общее число успешно закрытых сделок за период.
    *   Доход компании: Суммарная чистая прибыль агентства от всех продаж.
    *   Сумма продаж: Общая стоимость всей реализованной недвижимости.
    *   Средняя цена покупки: Средняя стоимость одного проданного объекта.
    *   Gross: Весь объем валовой комиссии до выплаты долей и расходов.
    *   Доходность в %: Процент чистого дохода относительно суммы продаж.
    *   Потеряно: Количество заявок, переведенных в отказ.
    *   Всего лидов: Общее число входящих заявок в базе.
    *   % Gross: Доля чистого дохода компании в валовой комиссии.
    *   Гистограмма: Наглядный рейтинг брокеров по объему чистой прибыли.
    *   Таблица рейтинга: Детальный план-факт активности и продаж по каждому сотруднику.
    
    #### **Анализ источников данных**
    | Показатель | Где берем СЕЙЧАС | Где берем в ИДЕАЛЕ | Комментарий |
    | :--- | :--- | :--- | :--- |
    | **К-во сделок** | amoCRM | **amoCRM** | Стандартный функционал CRM. |
    | **Сумма продаж** | amoCRM | **amoCRM** | Берем из поля "Цена юнита" в карточке сделки. |
    | **Доход компании**| amoCRM | **amoCRM** | Наша комиссия, прописанная в CRM. |
    | **Всего лидов** | amoCRM | **amoCRM** | Все входящие заявки из воронки. |
    | **Потеряно** | amoCRM | **amoCRM** | Все сделки в статусе «Закрыто и не реализовано». |
    | **ПЛАНЫ брокеров**| Таблица | **Внешняя система** | В амо нет места для хранения планов, используем таблицу. |
    | **ЗАТРАТЫ (Marketing)**| Таблица | **Внешняя система** | Чеки из FB/YT живут в таблице (это не сделки). |
    | **ЗАТРАТЫ (Invoices)**| Таблица | **Внешняя система** | Если инвойс не в Списках амо, таблица — лучший вариант. |
    | **Рейтинг (% вып.)**| Calculated | **amoCRM + Таблица** | Сравнение Факта (CRM) с Планом (Таблица). |

*   **Направления (Sales)**
    *   Департаменты: Off plan / Secondary / Rent.
    *   Финансы: GMV, Gross комиссия, Сплит брокеров и Net доход по каждому направлению отдельно.
    *   ROAS: Окупаемость затрат на маркетинг внутри каждого отдела продаж.
    *   *Примечание: Все данные по Support, Partnership и др. исключены из этого блока.*
    
    #### **Анализ источников данных**
    | Показатель | Где берем СЕЙЧАС | Где берем в ИДЕАЛЕ | Комментарий |
    | :--- | :--- | :--- | :--- |
    | **Департамент** | Таблица | **amoCRM (Поле)** | Требуется обязательное поле (Off plan/Secondary/Rent). |
    | **% Комиссии (Gross)**| Таблица/Голова | **amoCRM (Поле)** | Процент, который платит застройщик по данному лоту. |
    | **% Выплаты брокеру** | Таблица | **amoCRM (Поле)** | Индивидуальный % брокера в сделке (50/50, 60/40). |
    | **Чистый Доход** | Таблица | **Расчет (BI/CRM)** | Авто-расчет: (Сумма сделки * % Gross) - % Брокера. |
    | **Источник лида** | Таблица | **Авто-разметка (UTM)** | Система сама маркирует лид при входе из FB/Google/PF. |
    | **Затраты (Ads)** | Таблица | **Ads Connectors (API)** | FB, Google, PF тянут расходы в BQ напрямую. |
*   **План/Fact**
    *   Выполнение планов (KPI %): Процент реализации целей по сделкам, лидам и выручке.
    *   Личная воронка брокера: Анализ полученных (Received), закрытых (Won) и потерянных (Burned) лидов.
    *   Текущая загрузка (Active Leads): Количество клиентов в активной работе у каждого брокера.
    *   Дисциплина SLA: Среднее время ответа (Avg SLA) и объем первых контактов (First Touch).
    *   Глубокая аналитика отказов: Количественное и графическое распределение причин (Спам, Тест, Агент и т.д.).
    
    #### **Анализ источников данных**
    | Показатель | Где берем СЕЙЧАС | Где берем в ИДЕАЛЕ | Комментарий |
    | :--- | :--- | :--- | :--- |
    | **Плановые KPI** | Таблица | **Внешняя система** | Планы по лидам, сделкам и выручке за месяц. |
    | **Won / Burned** | amoCRM | **amoCRM** | Успешные и отказные статусы сделок. |
    | **Active Leads** | amoCRM | **amoCRM** | Текущие сделки в работе (не закрытые). |
    | **Причины отказа** | amoCRM | **amoCRM (Dropdown)** | Обязательный выбор из списка (Агент, Дубль, Спам и т.д.). |
    | **SLA / First Touch**| Events API | **API Событий** | Время создания лида и факт первого звонка/чата. |
    | **Revenue %** | Calculated | **CRM + Система** | Факт комиссии из CRM / План комиссии из Таблицы. |
*   **Брокер детали**
    *   Личные KPI за период: Fact Revenue (реальный доход) vs Plan Revenue (цель).
    *   История успеха (All-time): Total Deals (всего сделок) и Total Revenue (весь принесенный доход за историю).
    *   Персональный SLA: Средняя скорость ответа брокера на лид в минутах.
    *   Лог сделок: Таблица всех лидов брокера с названиями (lead_name) и ID.
    *   Аналитика касаний (Touches): Количество звонков и сообщений внутри каждой сделки.
    *   Deep Linking: Прямые ссылки (crm_url) на каждую сделку для быстрого доступа.
    
    #### **Анализ источников данных**
    | Показатель | Где берем СЕЙЧАС | Где берем в ИДЕАЛЕ | Комментарий |
    | :--- | :--- | :--- | :--- |
    | **All-time Stats** | Таблица | **amoCRM (История)** | Расчет всех успешных сделок по Broker ID за весь период. |
    | **SLA (SLA min)** | Не прозрачно | **API Событий** | Время первого контакта брокера с лидом из логов телефонии/WA. |
    | **Touches (Кол-во)** | Ручной ввод | **API Событий** | Автоматический подсчет всех событий (Звонки + Чаты) в ленте. |
    | **CRM Url** | Ручной ввод | **Авто-генерация** | Формирование ссылки по маске: `domain + lead_id`. |

### 2. Сделки компании
*   **Итоги (Global Summary)**
    *   **Глобальные KPI (Scorecards):**
        - Общее к-во сделок
        - Сумма сделок (GMV)
        - Комиссия (AED) (Gross)
        - Доход (AED) (Net)
    *   **Детализация по направлениям (Таблица):**
        - **Первичка:** income_pervichka, Средний чек (первичка), Размер комиссии, % брокера, Сумма сделки.
        - **Вторичка:** income_vtorichka, Средний чек (вторичка), Размер комиссии, % брокера, Сумма сделки.
        - **Аренда:** income_arenda, Средний чек (аренда), Размер комиссии, % брокера, Сумма сделки.
        - **Сопровождение:** income_support, Средний чек (супорт), gross_support, % брокера, Сумма сделки.
    *   **Графический анализ (Charts):**
        - Параметр "Категория/Отдел" (Pie Chart): Partnership, Support, Real Estate, CB, Rental, Primary, Secondary.
        - Рейтинг по "Record Count" (Bar Chart): Partners, Support, Sales (OP), Other, CB.
    
    #### **Анализ источников данных**
    | Показатель | Где берем СЕЙЧАС | Где берем в ИДЕАЛЕ | Комментарий |
    | :--- | :--- | :--- | :--- |
    | **income_[отдел]** | Таблица | **amoCRM** | Чистая прибыль агентства по конкретной воронке/направлению. |
    | **gross_[отдел]** | Таблица | **amoCRM** | Валовая комиссия до выплат по конкретной воронке. |
    | **Категории (CB, OP)**| Таблица | **amoCRM (Tags/Pipes)** | Маппинг всех типов сделок (CB - Commercial, OP - Off plan). |
    | **Record Count** | Calculated | **Calculated** | Количество строк (успешных сделок) в базе. |

*   **Рейтинг (Combined Leaderboard)**
    *   **Детализация сопровождения (Support Staff):**
        - Рейтинг сотрудников бек-офиса по количеству (Record Count) и доходности (Income) их сделок.
    *   **Глобальный рейтинг (Brokers & Partners):**
        - Сводная таблица эффективности всех участников процесса: внутренние брокеры + внешние агенты.
    *   **Визуальный анализ (Comparison):**
        - Использование Bar Charts прямо внутри ячеек таблицы для мгновенного сравнения Дохода (AED) и Объема сделок.
    
    #### **Анализ источников данных**
    | Показатель | Где берем СЕЙЧАС | Где берем в ИДЕАЛЕ | Комментарий |
    | :--- | :--- | :--- | :--- |
    | **Имя (Partner/Broker)**| Таблица | **amoCRM (Responsible)** | Имя ответственного или партнера из карточки сделки. |
    | **Доход (AED)** | Таблица | **amoCRM** | Чистая прибыль агентства (Net Income). |
    | **Record Count** | Calculated | **Calculated** | Общее количество успешно реализованных сделок. |

*   **Альянс (Drill-down & Partnerships)**
    *   **Интерактивный Реестр (Tabs):** Фильтрация по типам сделок (Off-plan, Rent, Secondary).
    *   **Воронка Партнера (Alliance Funnel):** Анализ не только денег, но и качества лидов (количество зашедших заявок vs количество закрытых сделок от конкретного партнера).
    *   **Финансовый Сплит:** Прозрачное отображение долей: `Gross` сделки -> `Выплата партнеру` -> `Доход агентства`.
    *   **Статус выплат (Payment Tracker):** Индикатор — выплачено вознаграждение внешнему агенту или оно находится в задолженности.
    *   **Drill-down (Глубокий аудит):** Клик на партнера открывает его «карточку здоровья» (конверсия, общая сумма выплат, список всех клиентов).
    
    #### **Анализ источников данных**
    | Показатель | Где берем СЕЙЧАС | Где берем в ИДЕАЛЕ | Комментарий |
    | :--- | :--- | :--- | :--- |
    | **Имя Партнера** | Таблица | **amoCRM (Field)** | Поле «Имя партнера» (Alliance Member Name). |
    | **Статус Выплаты** | Голова / Чат | **amoCRM (Dropdown)** | Статус: «Ожидает оплаты», «Выплачено», «Не выплачивается». |
    | **Конверсия** | Calculated | **amoCRM (История)** | Соотношение всех лидов партнера к его успешным сделкам. |
    | **Сумма Партнеру** | Таблица | **amoCRM (Field)** | Чистая сумма в AED, которую мы обязаны отдать партнеру. |

### 4. Инвойсы
*   **Инвойсы (Cash Flow Tracking)**
    *   **Short-term Индикаторы:** Мониторинг «быстрых» денег — сумма комиссии, количество счетов и чистый доход компании по ближайшим поступлениям.
    *   **Long-term Индикаторы:** Анализ отложенных платежей (рассрочки, этапы строительства) для долгосрочного планирования прибыли.
    *   **Реестр Инвойсов:** Детальная таблица с привязкой к конкретному Брокеру и Юниту (Название проекта + номер юнита).
    *   **Доход Компании (Net):** Фиксация финальной прибыли агентства (AED) по каждому выставленному инвойсу.
    
    #### **Анализ источников данных**
    | Показатель | Где берем СЕЙЧАС | Где берем в ИДЕАЛЕ | Комментарий |
    | :--- | :--- | :--- | :--- |
    | **Инвойсы / Счета** | Списки амо | **amoCRM (Lists)** | Используем системную сущность амо «Счета» (Invoices). |
    | **Short / Long** | Manual | **amoCRM (Tags/Field)** | Разделение по дате ожидаемого платежа или типу проекта. |

### 5. Маркетинг
*   **Маркетинг (Full Performance Audit)**
    *   **Qual Ratio (Чистая Квалификация):** Расчет процента целевых лидов с Исключением «Дублей» и «Тестовых» заявок.
    *   **Анализ причин потерь (Loss Audit):** Полное искоренение статуса «Не указана». Каждая отказная заявка ОБЯЗАНА иметь причину для корректировки рекламных кампаний.
    *   **Потенциальная ценность (Potential Value):** Суммарная стоимость всех лидов в воронке на базе поля «Бюджет». Позволяет видеть ROI ДО закрытия сделок.
    *   **Нормализация UTM:** Группировка кривых меток (напр. `{{ad.name}}`, `null`, `ad_1`) в единые понятные категории.
    *   **Lead Age (Pipeline Velocity):** Замер времени «застоя» — сколько дней лид не двигается по воронке, что является сигналом для ремаркетинга.
    
    #### **Анализ источников данных**
    | Показатель | Где берем СЕЙЧАС | Где берем в ИДЕАЛЕ | Комментарий |
    | :--- | :--- | :--- | :--- |
    | **Potential Value**| Лид-формы | **CRM (Estimated Budget)** | Брокер обязан ставить примерный бюджет на этапе Квалификации. |
    | **Loss Reason** | Пустота (null) | **CRM (Mandatory)** | Поле становится обязательным при переходе в «Закрыто-нерализовано». |
    | **UTM Campaign** | Брак ({{}}) | **Ads/CRM Logic** | Скрипт в BigQuery исправляет «кривые» метки на лету. |

### 6. ЛОГИЧЕСКАЯ СВЯЗКА (TECHNICAL GLUE)
Чтобы система работала автоматически и не требовала ручных таблиц-маппингов, внедряется следующая логика:

1.  **Унификация Полей (Field ID Mapping):**
    *   Разработчик BI-системы собирает ID всех финансовых полей и объединяет их в единые колонки.
2.  **Нормализация Источников (UTM Logic):**
    *   В amoCRM настраивается триггер (Salesbot): `utm_source` -> Поле `Источник`.
3.  **Скрипт: Розумне співпадіння (Fuzzy Matching):**
    *   Автоматическое сопоставление Имен Брокеров из таблиц планов с CRM через SQL-алгоритмы подобия (не боимся опечаток).
4.  **Скрипт: Фінансовий робот (Formula Webhook):**
    *   Авто-расчет поля `Net Income` = `Gross` * `% Broker Split` внутри amoCRM через вебхук для исключения ошибок ввода.
5.  **Скрипт: Наследование Планов (Auto-Goals):**
    *   Если на новый месяц план не установлен, BI подтягивает цели предыдущего месяца по умолчанию.
6.  **Защита от Дурака (Data Validation):**
    *   Запрет перехода в «Успех» без заполненных финансовых полей и валюты AED.

---
**Итог:** Настройка этой логики один раз заменяет десятки часов ручного ведения и маппинга таблиц ежемесячно.
